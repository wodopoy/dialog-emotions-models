# По какому критерию калибровать температуру: ECE или NLL

Раздел для диплома. Температурная калибровка подбирает один скаляр T, но *по какой
метрике* его минимизировать — открытый вопрос. Здесь — прямое сравнение двух
критериев отбора на всех 20 чекпоинтах. Источник:
`scripts/calibrate_objective_comparison.py` →
`artifacts/experiments/calibration/calibration_objective_comparison.csv`.

## Два критерия

- **ECE-fit** — T минимизирует **val ECE** (top-label калибровка, то, что мы репортим).
  Метрика бинированная, кусочно-постоянная, шумная.
- **NLL-fit** — T минимизирует **val NLL** (= кросс-энтропия = softmax-NLL). Гладкая,
  выпуклая по 1/T, **строго правильное правило оценки** (proper scoring rule): её
  минимум — это истинные вероятности. Классика Guo et al., 2017: фитят по NLL, репортят
  ECE. Для soft-меток NLL = KL + const, поэтому NLL-fit ≡ KL-fit.

Метод: T фитится per-model на **RuGo val** обоими критериями (free, без дедбанда, чтобы
изолировать эффект критерия), затем **ECE и KL** считаются на held-out **RuGo test** и
нативном **CEDR test** под raw (T=1) / T_ece / T_nll. ECE = столбик уверенности на
таймлайне; KL = верность всего 7-мерного вектора эмоций.

## Находки (17 не-вырожденных моделей)

| где | кто лучше по ECE | кто лучше по KL |
|---|---|---|
| **RuGo test** (in-domain) | **ECE-fit** (14 из 17) | NLL-fit (14/17) |
| **CEDR test** (деплой) | **NLL-fit** (12 из 17) | NLL-fit (14/17) |

1. **In-domain top-label ECE → выигрывает ECE-fit** (он напрямую оптимизирует этот прокси).
2. **Кросс-домен (деплой) ECE → выигрывает NLL-fit** (12/17). NLL фитит всё распределение
   и потому меньше переобучается под top-label бины RuGo и лучше переносится на нативный
   CEDR: ridge-char 0.045 → **0.031**, deberta 0.055 → **0.030**, tree-rf 0.070 → **0.041**.
3. **KL → выигрывает NLL-fit почти везде** (14/17), своим прямым критерием; по обоим
   доменам, малыми, но стабильными марджинами.
4. **Патология: NLL катастрофически пересмягчает «ненастоящие» логиты.** `fasttext` (ova,
   per-label скоры — не softmax-входы): T_nll = 1.42 → RuGo ECE **0.016 → 0.089**, CEDR
   **0.110 → 0.213**. `maxkazak` (несовпадение схемы меток): T_nll = 20 (потолок) → RuGo
   **0.39 → 0.25** хуже ECE-fit-евых 0.014. ECE-fit на этих моделях остаётся вменяемым.

## Рекомендация для диплома

**Записать ОБА исследования** (ECE-отбор + NLL-отбор + это сравнение — сравнение само по
себе результат), а **продакшн-калибровку делать по NLL с ECE-предохранителем**:

- **NLL-fit — рекомендуемый критерий.** Гладкий, proper scoring rule (защищается ссылкой на
  Guo et al.), не переобучается под шум бинов (дедбанд для него не нужен), даёт лучший KL
  всегда и лучший ECE на **деплой-домене** (нативный CEDR) для большинства моделей.
- **ECE-предохранитель.** После фита по NLL: если val ECE стал **хуже raw** — откатить к
  T = 1. Это автоматически ловит `fasttext`/`maxkazak` (их логиты не настоящие) и хранит
  честность столбика уверенности на таймлайне.
- **ECE-direct — наивный baseline.** Лучший для своей метрики in-domain, но (а) переобучается
  под шум бинов (потому и понадобился дедбанд), (б) хуже переносится кросс-домен. Его роль в
  работе — показать, *почему* нужен переход к NLL.

Размен честно: NLL чуть хуже **in-domain** (RuGo) ECE, но продукт деплоится на **нативном
русском** (CEDR-домен), где NLL лучше → это правильный размен. Для конкретного деплой-выбора
`logreg-char` разница мала в любом случае (он уже калиброван, raw CEDR 0.055; T_ece 0.049,
T_nll 0.064 — всё внутри SE ≈ 0.008), так что цена ошибки критерия для него почти нулевая;
критерий решает для `ridge-char`/трансформеров, где NLL заметно лучше на CEDR.

Альтернатива (если нужен один абзац, без «обоих исследований»): оставить ECE-отбор с
дедбандом как есть и приложить NLL как контроль — но тогда теряется кросс-доменный выигрыш
NLL, а это прямой довод про деплой. Предпочтительно — оба + переход на NLL.

## Воспроизведение

```bash
python scripts/calibrate_objective_comparison.py     # полный ECE+KL × raw/T_ece/T_nll
```

Примитивы: `best_temperature(objective="ece"|"nll")` и `negative_log_likelihood` в
`src/dialog_emo_models/metrics.py`. Основной прогон (`calibrate_temperature.py`) сейчас
отбирает по ECE+дедбанд и репортит `T_nll` как контроль — при переходе на рекомендацию его
надо переключить на NLL+ECE-guard (одна правка, по запросу).
